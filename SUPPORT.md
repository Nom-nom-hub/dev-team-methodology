# Support

If you need help with the Team-Driven Development methodology, here are several ways to get support:

## Documentation
- Read the main [README](README.md) for basic setup and usage
- Check the [Team-Driven Methodology Guide](team-driven.md) for detailed workflow information
- Review the [Team Personas Documentation](personas/) for specific role responsibilities

## GitHub Issues
- Search existing [issues](https://github.com/your-repo/dev-team-methodology/issues) to see if your question has already been answered
- Create a new issue for bugs, feature requests, or questions
- Tag your issue appropriately (bug, enhancement, question)

## Community
- Join our discussions in the GitHub repository
- Share your team methodology experiences and best practices

## For Specific Questions:

### Team Personas and Responsibilities
- Review the [Personas directory](personas/) for each role's definition and responsibilities
- Ensure all team members understand their specific inputs to each phase

### Command Usage
- Use `team --help` for CLI command information
- Check the [commands documentation](templates/commands/) for detailed command workflows

### Template Customization
- Templates are located in the [templates directory](templates/)
- Follow the existing format when creating custom templates

## When to Report Issues
- Bugs in the CLI tool or scripts
- Errors in team methodology documentation
- Issues with command templates or agent integration
- Suggestions for improving team collaboration workflows
- Problems with GitHub Actions workflows

## Response Time
We aim to respond to all issues within 3-5 business days. For urgent matters, please be patient as we are all volunteers working on this team methodology in our spare time.

Thank you for using Team-Driven Development!
